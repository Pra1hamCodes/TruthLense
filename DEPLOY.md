# Deployment Guide — Truth Lens

This project ships as **three** services. The frontend is a static SPA (deploy on Vercel); the backend is one Express API + one Flask ML sidecar (deploy on Render via the included `render.yaml`).

```
┌────────────┐   HTTPS    ┌────────────────┐   HTTP    ┌─────────────────────┐
│  Vercel    │ ─────────▶ │  Render        │ ────────▶ │ Render: Flask analyzer │
│  (Vite SPA)│            │  Express :3000 │           │ (CPU PyTorch ensemble) │
└────────────┘            └────────────────┘           └───────────────────────┘
```

## 0. Prerequisites

- A MongoDB Atlas cluster (free M0 works). Whitelist `0.0.0.0/0` so Render can reach it.
- Git LFS installed locally (`git lfs install`) — only needed if you push large model weights yourself.
- Free accounts on **Render** and **Vercel** linked to your GitHub.

## 1. Push to GitHub (already done if you cloned this repo)

The repository can track large model weights via Git LFS if you choose to commit them. Render's build environment fetches LFS objects automatically.

## 2. Deploy backend + Flask servers (Render)

1. Render dashboard → **New + → Blueprint** → connect this GitHub repo.
2. Render reads `render.yaml` and provisions two services:
   - `truth-lens-backend` (Node, port 10000)
   - `truth-lens-flask-vit` (Python, port auto)
3. Fill in the secrets Render asks for (these are marked `sync: false` in `render.yaml`):

   | Service | Variable | Example value |
   |---|---|---|
   | `truth-lens-backend` | `MONGO_URI` | `mongodb+srv://USER:PASS@cluster0.xxxx.mongodb.net/truth-lens?retryWrites=true&w=majority` |
   | `truth-lens-backend` | `CORS_ORIGINS` | `https://your-app.vercel.app,http://localhost:5173` |
   | `truth-lens-backend` | `FLASK_VIT_URL` | `https://truth-lens-flask-vit.onrender.com` |
   | `truth-lens-flask-vit` | `FLASK_CORS_ORIGINS` | `https://truth-lens-backend.onrender.com` |

   `JWT_SECRET` is auto-generated by Render. Update `FLASK_VIT_URL` *after* the Flask service comes up so you have its public URL.

4. First Flask deploy is slow (~10–15 min) because it pulls PyTorch, facenet-pytorch, and the ensemble weights. Subsequent deploys are cached.

## 3. Deploy frontend (Vercel)

1. Vercel dashboard → **Add New… → Project** → import this repo.
2. **Root Directory**: `frontend`
3. Framework preset is autodetected as **Vite**.
4. Add environment variables (Settings → Environment Variables):

   | Variable | Value |
   |---|---|
   | `VITE_API_BASE_URL` | `https://truth-lens-backend.onrender.com` |
   | `VITE_FLASK_MAIN_URL` | `https://truth-lens-flask-vit.onrender.com` |
   | `VITE_FLASK_OLD_URL` | `https://truth-lens-flask-vit.onrender.com` |

5. Deploy. After the first build, copy the Vercel URL (e.g. `https://your-app.vercel.app`) back into the Render `CORS_ORIGINS` for `truth-lens-backend` and the `FLASK_CORS_ORIGINS` of the Flask service, then redeploy it.

## 4. Smoke test

```
curl https://truth-lens-backend.onrender.com/api/health      # → {"status":"ok"} (or similar)
curl https://truth-lens-flask-vit.onrender.com/              # 404 expected; server responds
```

Open the Vercel URL → Sign up → Upload a video → wait for analysis. The first request after a cold start can take ~60 s on Render's free tier (services spin down when idle).

## 5. Local development

Backend:
```
cd backend
cp .env.example .env     # fill in MONGO_URI + JWT_SECRET
npm install
npm start                # http://localhost:3000
```

Flask analyzer:
```
cd backend/flask_server_2    # ViT (.pt, ships via Git LFS)
python -m venv .venv && .venv/Scripts/activate
pip install -r requirements.txt
python app.py                 # http://localhost:5001
```

Frontend:
```
cd frontend
cp .env.example .env
npm install
npm run dev                  # http://localhost:5173
```

## 6. Notes on model weights

- `backend/flask_server_2/models/efficientnet_b4.pt`, `vit_b16.pt`, `freqnet_r18.pt`, and `ensemble_stacker.pkl` are the analyzer artefacts expected at runtime.
- Render free tier has 512 MB RAM. The ensemble is CPU-only, but the sidecar still needs enough memory for three checkpoints and MTCNN.
