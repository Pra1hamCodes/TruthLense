# Deployment Guide — DeepShield

This project ships as **four** services. The frontend is a static SPA (deploy on Vercel); the backend is one Express API + two Flask ML sidecars (deploy on Render via the included `render.yaml`).

```
┌────────────┐   HTTPS    ┌────────────────┐   HTTP    ┌─────────────────────┐
│  Vercel    │ ─────────▶ │  Render        │ ────────▶ │ Render: Flask CNN   │
│  (Vite SPA)│            │  Express :3000 │           │ (TensorFlow .h5)    │
└────────────┘            │                │ ────────▶ │ Render: Flask ViT   │
                          │                │           │ (PyTorch .pt LFS)   │
                          └────────────────┘           └─────────────────────┘
```

## 0. Prerequisites

- A MongoDB Atlas cluster (free M0 works). Whitelist `0.0.0.0/0` so Render can reach it.
- Git LFS installed locally (`git lfs install`) — only needed if you push more model weights yourself.
- Free accounts on **Render** and **Vercel** linked to your GitHub.

## 1. Push to GitHub (already done if you cloned this repo)

The repository tracks `*.pt` model weights via Git LFS. Render's build environment fetches LFS objects automatically.

## 2. Deploy backend + Flask servers (Render)

1. Render dashboard → **New + → Blueprint** → connect this GitHub repo.
2. Render reads `render.yaml` and provisions three services:
   - `deepshield-backend` (Node, port 10000)
   - `deepshield-flask-cnn` (Python, port auto)
   - `deepshield-flask-vit` (Python, port auto)
3. Fill in the secrets Render asks for (these are marked `sync: false` in `render.yaml`):

   | Service | Variable | Example value |
   |---|---|---|
   | `deepshield-backend` | `MONGO_URI` | `mongodb+srv://USER:PASS@cluster0.xxxx.mongodb.net/deepshield?retryWrites=true&w=majority` |
   | `deepshield-backend` | `CORS_ORIGINS` | `https://your-app.vercel.app,http://localhost:5173` |
   | `deepshield-backend` | `FLASK_CNN_URL` | `https://deepshield-flask-cnn.onrender.com` |
   | `deepshield-backend` | `FLASK_VIT_URL` | `https://deepshield-flask-vit.onrender.com` |
   | `deepshield-flask-cnn` | `FLASK_CORS_ORIGINS` | `https://deepshield-backend.onrender.com` |
   | `deepshield-flask-vit` | `FLASK_CORS_ORIGINS` | `https://deepshield-backend.onrender.com` |

   `JWT_SECRET` is auto-generated by Render. Update `FLASK_*_URL` *after* the Flask services come up so you have their public URLs.

4. First Flask deploy is slow (~10–15 min) because it pulls TensorFlow / PyTorch wheels and the LFS-tracked `as_model_0.837.pt` (~214 MB). Subsequent deploys are cached.

## 3. Deploy frontend (Vercel)

1. Vercel dashboard → **Add New… → Project** → import this repo.
2. **Root Directory**: `frontend`
3. Framework preset is autodetected as **Vite**.
4. Add environment variables (Settings → Environment Variables):

   | Variable | Value |
   |---|---|
   | `VITE_API_BASE_URL` | `https://deepshield-backend.onrender.com` |
   | `VITE_FLASK_MAIN_URL` | `https://deepshield-flask-cnn.onrender.com` |
   | `VITE_FLASK_OLD_URL` | `https://deepshield-flask-vit.onrender.com` |

5. Deploy. After the first build, copy the Vercel URL (e.g. `https://your-app.vercel.app`) back into the Render `CORS_ORIGINS` for `deepshield-backend` and the `FLASK_CORS_ORIGINS` of both Flask services, then redeploy them.

## 4. Smoke test

```
curl https://deepshield-backend.onrender.com/api/health      # → {"status":"ok"} (or similar)
curl https://deepshield-flask-cnn.onrender.com/              # 404 expected; server responds
curl https://deepshield-flask-vit.onrender.com/              # 404 expected; server responds
```

Open the Vercel URL → Sign up → Upload a video → wait for analysis. The first request after a cold start can take ~60 s on Render's free tier (services spin down when idle).

## 5. Local development

Backend:
```
cd backend
cp .env.example .env     # fill in MONGO_URI + JWT_SECRET
npm install
npm start                # http://localhost:3000
```

Flask servers (in separate terminals):
```
cd backend/flask_server      # CNN+LSTM (.h5, ships in git)
python -m venv .venv && .venv/Scripts/activate
pip install -r requirements.txt
python app.py                 # http://localhost:5000

cd backend/flask_server_2    # ViT (.pt, ships via Git LFS)
python -m venv .venv && .venv/Scripts/activate
pip install -r requirements.txt
python app.py                 # http://localhost:5001
```

Frontend:
```
cd frontend
cp .env.example .env
npm install
npm run dev                  # http://localhost:5173
```

## 6. Notes on model weights

- `backend/flask_server/models/deepfake_detection_model.h5` (~95 MB) — committed directly.
- `backend/flask_server_2/as_model_0.837.pt` (~214 MB) — tracked via **Git LFS** (`*.pt` filter in `.gitattributes`). Free GitHub LFS gives 1 GB storage / 1 GB bandwidth per month. Heavy redeploys may exhaust this; if you hit the cap, host the `.pt` on Hugging Face Hub or S3 and download it in `flask_server_2`'s build step instead.
- Render free tier has 512 MB RAM. The ViT model is borderline — if it OOMs, upgrade that single service to the $7 Starter plan.
